# Проект автоматизации тестирования интернет-магазина «Капибара»

## 📌 Описание проекта

Проект представляет собой набор автотестов для интернет-магазина [капибара161.рф](https://капибара161.рф).  
Тесты покрывают:

- Полный бизнес-путь пользователя: выбор товара с фильтрами (сортировка по цене, алфавиту, диапазон цен), добавление в корзину, оформление заказа, проверка финальной цены.
- Негативные сценарии (невалидные данные при оформлении).
- Операции с корзиной: увеличение/уменьшение количества, удаление товара, проверка пересчёта суммы.
- Мониторинг цен (проверка, что цена положительна, а также оповещение о выходе за рыночные границы).
- Проверка наличия товара на странице каталога и конкретного товара.

Проект реализован с использованием **Page Object Model (POM)**, **PyTest**, **Allure** для отчётов, **Selenium WebDriver** (Firefox локально, Chrome в CI), а также **requests + BeautifulSoup** для быстрого парсинга цен.

## 🚀 Запуск тестов

### Локальный запуск (Firefox, с графическим окном)

По умолчанию тесты используют Firefox без headless (видно окно браузера).  
Для этого достаточно:

```bash
pytest tests/ -v -s
```

Чтобы запустить конкретный тест:

```bash
pytest tests/test_buy_product.py::TestBuyProduct::test_buy_product_1 -v -s
```

### Локальный запуск с Chrome (headless)

```bash
BROWSER=chrome pytest tests/ -v -s
```

### Запуск в CI (GitHub Actions)

В репозитории уже настроен workflow `.github/workflows/tests.yml`. При каждом пуше в ветку `main` или по расписанию тесты запускаются с **Chrome в headless-режиме**, генерируется Allure-отчёт и сохраняется как артефакт.  
Результаты можно посмотреть во вкладке **Actions** → выбрать запуск → скачать `allure-report`.

## 🔧 Особенности реализации

- **Универсальный парсинг цены**: сначала `requests` (быстро), если не удалось – `Selenium` (надёжно). Это позволяет обходить простую JS-защиту.
- **Проверки цен** в тестах: сравнение цены товара в каталоге с ценой в корзине, проверка суммы двух товаров, корректность пересчёта при изменении количества.
- **Работа с корзиной**: отдельные методы в `CartPage` для получения количества, цены за единицу, общей суммы, кликов по плюсу/минусу, удаления с подтверждением.
- **Обработка `StaleElementReferenceException`** в базовом классе: повторные попытки для кликов и ввода текста.
- **Поддержка Firefox и Chrome** через переменную окружения `BROWSER`. Локально – Firefox (ваши локаторы), в CI – Chrome (стабильнее в headless).
- **Allure-шаги и вложения**: скриншоты при падении, логи, информация о ценах.
- **Логирование**: каждый шаг записывается в файл с меткой времени.

## 📁 2. Создание файла `.gitignore`

Чтобы `venv` и другие временные/служебные файлы не попадали в репозиторий, создайте в корне проекта файл `.gitignore` со следующим содержимым:

```gitignore
# Виртуальное окружение
venv/
env/
ENV/
.env/

# Кэш Python
__pycache__/
*.pyc
*.pyo
*.pyd

# Логи и отчёты
logs/
allure-results/
allure-report/
screenshots/
*.log

# Файлы с ценами (кэш)
price_cache.txt
price_history_*.csv

# Настройки IDE
.vscode/
.idea/
*.iml

# OS метаданные
.DS_Store
Thumbs.db

# Файлы, сгенерированные при тестах
.geckodriver/
chromedriver
geckodriver
```

После добавления этого файла выполните команды, чтобы исключить уже отслеживаемые файлы (если venv уже закоммичен):

```bash
git rm -r --cached venv
git add .gitignore
git commit -m "Add .gitignore and remove venv from tracking"
git push
```
